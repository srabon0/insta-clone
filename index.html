A session is something when a user use an app or website for a particular amount of time. Webstorage api is used by the developers to stroe some data in web browser and there are two mechanism for that one is session stroage and another is loacl storage.
For session storage
The data is stored in persisted amount of time , suppose a user browse a web or using an app, now some data is stored in the browser and as soon as the user closes the particular tab or window or the app the session is end and the data stroed in the browser is lost. This is session storage. In Session storage stroed data is not sent to the server or no network request is called using this data.It has more capacity than cookies.
On the other hand
Local storage is the same as session storage but the major difference is the data stored in the local storage , it doesn't clear itself which means the data is stored or saved in the browser , it is stored even if the user closes the tab, window, close the app or shut down the system. Local stroage data is used for actually store some user relevant data which is not that important to be saved on the server. For the fast and optimized performance it is stored in the browser so that user doesnt have to network call request or call an api from the server everytime the user need reusable data. Local storage has more capacity than session storage or cookies.
